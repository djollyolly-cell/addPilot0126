# Реферальная система AdPilot

## Дата: 2026-04-07

## Цель

Привлечение новых пользователей и удержание текущих через реферальные бонусы.

---

## Механика

- У каждого пользователя есть **уникальный реферальный код** (генерируется автоматически при регистрации)
- Два типа ссылок (настраивается админом):
  - **Обычная** — бонус только рефереру
  - **Со скидкой** — приглашённый получает скидку на первую оплату (по умолчанию 10%, редактируемая через админку)
- Реферал засчитывается при **первой оплате** приглашённого
- Приглашённый вводит реф-код при оплате (поле рядом с промокодом в PaymentModal)

---

## Бонусы рефереру

| Условие | Бонус |
|---|---|
| Каждый оплативший реферал | +7 дней к текущей подписке |
| 3 оплативших реферала (суммарно) | +30 дней (бесплатный месяц) |
| 10+ оплативших рефералов (суммарно) | Скидка 15% на следующую оплату |

Бонус +7 дней начисляется за каждого. Бонус +30 дней при достижении 3 — разовый, суммируется с обычными +7. Скидка 15% при 10+ — постоянная на все последующие оплаты.

---

## Бонусы приглашённому

| Тип ссылки | Бонус |
|---|---|
| Обычная | Ничего |
| Со скидкой | Скидка на первую оплату (по умолчанию 10%, настраивается per-code через админку) |

---

## Уведомления

При первой оплате приглашённого — рефереру приходит Telegram-сообщение:
> 🎁 По вашему промокоду подключился новый пользователь! Вам начислено +7 дней к подписке. Всего рефералов: X.

При достижении 3 рефералов — дополнительное сообщение:
> 🎉 3 реферала! Вам начислен бесплатный месяц (+30 дней).

При достижении 10 рефералов:
> 🏆 10 рефералов! Теперь вы получаете скидку 15% на все оплаты.

## Схема данных

### Новые поля в `users`

```
referralCode: string          — уникальный код (REF-XXXXXX)
referralType: "basic" | "discount"  — тип ссылки (по умолчанию "basic")
referralDiscount: number      — скидка для приглашённых, % (по умолчанию 10)
referralCount: number         — количество оплативших рефералов
referralBonusDaysEarned: number — всего дней заработано через рефералов
referredBy: Id<"users">       — кто привёл этого пользователя (optional)
referralMilestone3Claimed: boolean  — бонус за 3 реферала уже начислен
referralMilestone10Reached: boolean — достигнут порог 10 рефералов
```

### Новая таблица `referrals`

```
referrerId: Id<"users">       — кто пригласил
referredId: Id<"users">       — кого пригласили
referralCode: string          — использованный код
status: "registered" | "paid" — статус реферала
paymentId: Id<"payments">     — ID оплаты (при status=paid)
bonusDaysGranted: number      — сколько дней начислено рефереру
createdAt: number             — дата регистрации
paidAt: number                — дата первой оплаты (optional)

Indexes: by_referrerId, by_referredId, by_referralCode
```

### Изменения в `payments`

```
referralCode: string (optional) — реферальный код, введённый при оплате
referralDiscount: number (optional) — применённая скидка, %
```

## UI — Вкладка "Рефералы" в Настройках

Новая вкладка рядом с Профиль, Телеграм, API, Бизнес.

### Содержимое:

1. **Реферальный код** — отображение кода + кнопка "Копировать"
2. **Статистика**:
   - Всего приглашённых (зарегистрированных)
   - Оплативших
   - Дней заработано
3. **Прогресс-бары**:
   - До бесплатного месяца: X/3 оплативших
   - До скидки 15%: X/10 оплативших
4. **Список рефералов** — таблица: имя/email, дата, статус (зарегистрирован / оплатил)

---

## UI — PaymentModal (ввод кода)

- Поле ввода реферального кода рядом с полем промокода
- При вводе валидного кода со скидкой — показать применённую скидку
- Реферальный код и промокод могут использоваться одновременно (скидки суммируются)
- Валидация: код существует, тип "discount", пользователь не может использовать свой код

---

## Админка

- Раздел "Рефералы":
  - Список всех рефереров: имя, email, количество рефералов, тип ссылки, скидка
  - Редактирование per-user: тип ссылки (basic/discount), размер скидки (%)
  - Фильтр: по количеству рефералов, по типу

---

## Привязка реферала — Flow

1. Приглашённый вводит реф-код в PaymentModal при первой оплате
2. Backend валидирует код (`validateReferralCode` query)
3. Если тип "discount" — применяет скидку к сумме оплаты
4. При успешной оплате (`completePayment` mutation):
   - Создаёт запись в `referrals` со статусом "paid"
   - Начисляет рефереру +7 дней (`subscriptionExpiresAt += 7 days`)
   - Увеличивает `referralCount` реферера
   - Проверяет milestone 3 → +30 дней
   - Проверяет milestone 10 → ставит флаг
   - Отправляет Telegram-уведомление рефереру
   - Сохраняет `referralCode` и `referralDiscount` в payments

---

## Генерация кода

Формат: `REF-XXXXXX` (6 символов, uppercase буквы + цифры).
Генерируется автоматически при создании пользователя.
Уникальность проверяется через index `by_referralCode` на таблице users.

---

## Ограничения

- Пользователь не может использовать свой собственный код
- Реферальный код можно использовать только при **первой** оплате
- Один приглашённый = один реферер (поле `referredBy` write-once)
- Скидка 15% при 10+ рефералах — постоянная, применяется автоматически при оплате
