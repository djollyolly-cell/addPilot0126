# AI Кабинет — Спецификация (v2)

## Обзор
Страница "AI Кабинет" для создания и управления рекламными кампаниями VK Ads через AI. Пользователь заполняет минимум полей, AI формирует объявления автоматически.

## Целевая аудитория
- Владельцы бизнеса / маркетологи — не хотят разбираться в 100 настройках myTarget
- Таргетологи — хотят быстро запускать кампании с AI-помощником

## Навигация
- Новый пункт в сайдбаре: **"AI Кабинет"** (иконка Sparkles)
- Между "Кабинеты" и "Правила" в порядке навигации
- Три страницы: `/ai-cabinet` (хаб), `/ai-cabinet/new` (создание), `/ai-cabinet/:id` (детали)

---

## Структура данных myTarget (ИСПРАВЛЕНО)

> **Важно:** В myTarget API **нет** сущности "группа объявлений". Структура плоская:

```
Кампания (campaign)
├── Баннер 1 (banner) = одно объявление
├── Баннер 2 (banner) = одно объявление
└── Баннер 3 (banner) = одно объявление
```

- **Кампания** — верхний уровень. Содержит: название, package (формат/цель), таргетинги, бюджет, ставка
- **Баннер** — одно объявление внутри кампании. Содержит: текстовые блоки, контент (изображения/видео), URL
- **Бюджет** задаётся на уровне **кампании** (`budget_limit_day`), НЕ баннера
- **Таргетинги** (гео, пол, возраст) задаются на уровне **кампании**
- **Package** определяет формат объявления, доступные placements и типы оплаты

## Страница 2: Создание кампании `/ai-cabinet/new`

### Формат: Аккордеон
Секции раскрываются по порядку. Заполнил первую → раскрывается вторая.

### Шаг 1 — О бизнесе
| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| Направление бизнеса | Text input | Да | "Стоматология", "Фитнес" и т.д. |
| Ссылка | URL input | Да | Сайт, профиль/сообщество ВК, лид-форма |

### Шаг 2 — Настройки кампании
| Поле | Тип | Обязательное | Описание |
|---|---|---|---|
| Цель рекламы | Select | Да | См. таблицу целей ниже |
| Гео | Autocomplete регионов | Да | Поиск по названию → region_id из API |
| Возраст | Range (от — до) | Нет | Дефолт: 22–55. Диапазон 12–75 |
| Пол | Radio: Все / М / Ж | Нет | Дефолт: Все. API: "MF" / "M" / "F" |
| Дневной бюджет | Number input | Да | В рублях. Мин: 100₽ |

#### Цели рекламы → myTarget package mapping
| Цель в UI | Описание | Package | Objective |
|---|---|---|---|
| Трафик на сайт | Клики → сайт | `multiformat` | traffic |
| Вступления в сообщество | Подписки ВК | `social_engagement` | social |
| Сообщения сообщества | Лиды через ЛС | `lead_ads` | messages |
| Продвижение видео | Просмотры видео | `video_multiformat` | video_views |
| Продвижение поста | Промо публикации | `post_promo` | engagement |

> **Примечание:** точные package_id загружаются из `GET /api/v2/packages.json` при первом использовании и кешируются.

#### Гео-таргетинг
- Источник данных: `GET /api/v2/regions.json` — иерархическое дерево (страна → регион → город)
- UI: autocomplete с поиском по названию города/региона
- Множественный выбор (можно несколько городов)
- Кешируем дерево регионов в `_storage` (обновляем раз в неделю)

### Шаг 3 — Объявления (AI)
- AI автоматически генерирует **3 варианта** баннеров для A/B тестирования
- Генерация на основе: направление + цель + ссылка (AI парсит страницу)
- Каждый баннер содержит:

| Поле | Лимит | Описание |
|---|---|---|
| Заголовок (title) | 25 символов | Краткий заголовок |
| Текст (text) | 90 символов | Основной текст объявления |
| Изображение 600×600 | обязательное | Квадрат для мультиформата |
| Изображение 1080×607 | обязательное | Широкий формат |
| Иконка 256×256 | обязательное | Для нативных размещений |

- Чекбоксы: пользователь выбирает какие баннеры запустить (минимум 1)
- Кнопка "✨ Перегенерировать" — новый вариант
- Кнопка "✏️ Редактировать" — ручное редактирование текста
- Кнопка ✨ рядом с каждым текстовым полем — AI-подсказка

> **Генерация изображений:** используем существующий `creatives.generateImage` (FLUX API через BFL.ai). Генерируем 600×600, масштабируем для 1080×607 и 256×256 если нужно.

### Шаг 4 — Превью и запуск
- Превью всех настроек (кампания + таргетинги + выбранные баннеры)
- Прогноз аудитории (опционально): `GET /api/v2/projection.json` — показывает размер аудитории по таргетингам
- Кнопка "🚀 Запустить кампанию"
- Порядок API-вызовов при запуске:
  1. `POST /api/v2/content/static.json` × N — загрузить изображения (600×600, 1080×607, 256×256)
  2. `POST /api/v2/campaigns.json` — создать кампанию с таргетингами и бюджетом
  3. `POST /api/v2/banners.json` × N — создать баннеры с `content` + `textblocks`
- После запуска: редирект на `/ai-cabinet/:id`

- Кнопка "← Назад" (к хабу)
- Название кампании
- Бейдж статуса
- Кнопки: "⏸ Пауза" / "▶ Запустить", "🗑 Удалить"

### Метрики (6 карточек)
Расход | Лиды | CPL | CTR | Показы | Клики

### Две колонки

**Левая колонка:**
1. **Настройки кампании** — цель, направление, ссылка, гео, возраст, пол, бюджет + кнопка "Редактировать"
2. **Модерация баннеров** — статус каждого баннера:
   - `allowed` → "Одобрено" (зелёный)
   - `new` / `changed` / `delayed` → "На модерации" (жёлтый)
   - `banned` → "Отклонено" (красный) + причина + кнопка "✨ Перегенерировать"
3. **Последние события (лог)** — "Получен лид", "Кампания запущена", "Баннер одобрен" с таймстемпами

**Правая колонка:**
1. **Баннеры** — превью активных баннеров (изображение + текст). Кнопки: "✨ Перегенерировать", "✏️ Редактировать", "⏸ Остановить"
2. **AI рекомендации** — "CPL выше целевого, попробуйте расширить гео" + кнопки "Применить" / "Отклонить"

## AI-помощник

### Генерация объявлений
- Вход: направление бизнеса + цель + ссылка (парсинг страницы через fetch для контекста)
- Выход: title (≤25), text (≤90), 3 изображения (600×600, 1080×607, 256×256)
- Генерирует 3 варианта для A/B тестирования
- Используем существующие: `creatives.generateText` (Claude) + `creatives.generateImage` (FLUX)

### Кнопка ✨ в полях
- Рядом с текстовыми полями при редактировании баннера
- AI генерирует/улучшает текст конкретного поля (с учётом лимита символов)

### AI рекомендации (на странице деталей)
- Анализ метрик кампании (запускается по крону раз в 6 часов для активных кампаний)
- Типы рекомендаций:
  - `increase_budget` — CPL ниже целевого, масштабируем
  - `decrease_budget` — бюджет сливается без лидов
  - `pause_banner` — конкретный баннер неэффективен
  - `expand_geo` — маленькая аудитория, добавить города
  - `regenerate` — CTR ниже среднего, нужны новые креативы
- Кнопки "Применить" (выполнить через API) / "Отклонить"
- Уведомление в Telegram при новой рекомендации

### Обработка модерации
- Polling статуса баннеров каждые 5 мин (вместе с syncMetrics)
- При `banned`: уведомление в Telegram + предложение перегенерировать
- При `allowed`: обновить статус в БД

## Технические детали

### myTarget API v2 — используемые эндпоинты

| Действие | Метод | Эндпоинт | Примечания |
|---|---|---|---|
| Список packages | GET | `/api/v2/packages.json` | Кешировать, определяет формат/цель |
| Дерево регионов | GET | `/api/v2/regions.json` | Кешировать, для гео-таргетинга |
| Прогноз аудитории | GET | `/api/v2/projection.json` | Опционально, превью перед запуском |
| Загрузить изображение | POST | `/api/v2/content/static.json` | FormData: file + width + height |
| Создать кампанию | POST | `/api/v2/campaigns.json` | name, package, targetings, budget |
| Создать баннер | POST | `/api/v2/banners.json` | campaign_id, content, textblocks |
| Обновить кампанию | POST | `/api/v2/campaigns/{id}.json` | Статус, бюджет, таргетинги |
| Обновить баннер | POST | `/api/v2/banners/{id}.json` | Статус, контент, текст |
| Получить баннеры | GET | `/api/v2/banners.json` | С полем moderation_status |
| Статистика | GET | `/api/v2/statistics/...` | Существующий getMtStatistics |

## Зависимости от существующего
- `adAccounts` — выбор аккаунта + `mtAdvertiserId` + `accessToken`
- `vkApi.ts` — `callMtApi()` для запросов + существующие `getMtBanners`, `getMtStatistics`
- `syncMetrics` — метрики AI-кампаний подхватываются автоматически (через vkCampaignId)
- `ruleEngine` — правила мониторинга работают и для AI-кампаний
- `creatives.generateText` / `generateImage` — переиспользуем логику генерации (Claude + FLUX)
- `telegram.ts` — уведомления о модерации и рекомендациях

## Известные ограничения
1. **Package ID** — зависит от аккаунта. Нужно загрузить `packages.json` и найти нужный по типу
2. **Pads** (площадки) — обязательное поле в таргетингах, берётся из package
3. **Лимиты символов** — title ≤25, text ≤90 для мультиформата. AI должен строго соблюдать
4. **Модерация** — занимает от минут до часов. Баннер не показывается пока не одобрен
5. **Минимальный бюджет** — зависит от формата, обычно от 100₽/день
6. **Изображения**: `content/static.json` требует точные размеры (600×600, 1080×607, 256×256)
