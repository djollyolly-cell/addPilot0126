# Профиль бизнеса — Design Spec

**Дата:** 2026-03-28
**Скоуп:** Профиль бизнеса для аккаунта + направления бизнеса + интеграция в AI-промпты

---

## Проблема

AI-генерация текстов, изображений и анализ видео работают без контекста бизнеса. Промпты не знают нишу, аудиторию, тон бренда — результаты получаются generic и требуют ручной доработки. Для будущего автоматического запуска кампаний профиль бизнеса — обязательный фундамент.

## Решение

### 1. Хранение данных

**Расширение таблицы `adAccounts`** (новые optional поля):

| Поле | Тип | Описание |
|------|-----|----------|
| `companyName` | `v.optional(v.string())` | Название компании |
| `industry` | `v.optional(v.string())` | Ниша/отрасль |
| `tone` | `v.optional(v.string())` | Тон коммуникации: `friendly` / `professional` / `expert` / пользовательский |
| `website` | `v.optional(v.string())` | URL сайта |

**Новая таблица `businessDirections`**:

| Поле | Тип | Описание |
|------|-----|----------|
| `accountId` | `v.id("adAccounts")` | Привязка к аккаунту |
| `name` | `v.string()` | Название направления (напр. "Доставка еды") |
| `targetAudience` | `v.optional(v.string())` | ЦА (может быть сгенерирована AI) |
| `usp` | `v.optional(v.string())` | УТП (может быть сгенерирована AI) |
| `isActive` | `v.boolean()` | Активно ли направление |
| `createdAt` | `v.number()` | Дата создания |

Индекс: `by_accountId`.

### 2. UI — редактирование профиля

**Общий компонент `BusinessProfileEditor`** — переиспользуется в двух местах:

**Страница Кабинеты** — при раскрытии карточки аккаунта, секция "Профиль бизнеса":
- 4 поля: Название компании, Ниша, Тон (select: Дружелюбный / Деловой / Экспертный / Своё), Сайт
- Кнопка "Сохранить профиль"
- Блок "Направления бизнеса":
  - Список существующих направлений (название + ЦА + УТП в компактном виде)
  - Кнопка "Добавить направление" → инлайн-форма
  - Рядом с полями ЦА и УТП — кнопка Sparkles для AI-генерации вариантов
  - Редактирование / удаление направлений

**Страница Настройки** — та же секция для активного аккаунта через тот же компонент `BusinessProfileEditor`.

### 3. Выбор направления при генерации

На страницах CreativesPage и VideosPage (при генерации):
- Если у аккаунта есть направления — дропдаун "Направление" перед генерацией
- Если одно направление — выбирается автоматически
- Кнопка "Добавить направление" рядом с дропдауном — для быстрого создания нового
- Если нет направлений — генерация работает как сейчас (без бизнес-контекста)

### 4. AI-подсказки для ЦА и УТП

**Механизм:** По кнопке Sparkles рядом с полями ЦА и УТП в форме направления.

**Промпт получает:** название компании, ниша, название направления.

**Результат:** 3-5 вариантов в виде кликабельного списка. Пользователь выбирает вариант — он вставляется в поле. Можно отредактировать вручную после выбора.

**Бэкенд:** Новый action `businessDirections.suggestTargetAudience` и `businessDirections.suggestUsp`, вызывают Claude API с контекстом бизнеса.

### 5. Интеграция в AI-промпты

**Генерация текстов** (`convex/creatives.ts:generateText`):
В system prompt добавляется блок бизнес-контекста:
```
Контекст бизнеса:
- Компания: {companyName}
- Ниша: {industry}
- Тон: {tone}
- Направление: {directionName}
- Целевая аудитория: {targetAudience}
- УТП: {usp}
```
Если данных нет — промпт работает как сейчас (без бизнес-контекста).

**Генерация изображений** (`convex/creatives.ts:generateImage`):
В DALL-E промпт автоматически добавляется бизнес-контекст (бренд, ниша, направление) для более релевантного стиля. Пользователь не видит эту информацию — подтягивается автоматически из профиля.

**Анализ видео** (`convex/creativeAnalytics.ts:analyzeWatchRates`):
В system prompt добавляется бизнес-контекст для более точных рекомендаций (напр. "для доставки еды важно показать еду в первые 3 секунды").

## Что НЕ входит в этот скоуп

- Автоматический запуск кампаний (отдельный будущий скоуп, профиль — фундамент)
- Загрузка логотипа / брендбука
- Шаблоны креативов по нишам
- Сравнение эффективности по направлениям
