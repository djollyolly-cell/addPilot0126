AI Кабинет — Архитектура страницы

3 подхода к организации полного хаба

A

Одна страница с табами

/ai-cabinet — единая страница с табами: Создать | Кампании | Аналитика

Выбор кабинета — dropdown сверху. Всё на одной странице, переключение между разделами через табы.

Плюсы

  • Всё в одном месте, быстрая навигация
  • Один пункт в сайдбаре
  • Проще переключаться между созданием и списком

Минусы

  • Страница может стать перегруженной
  • Табы внутри табов если детали кампании тоже с табами
B

Хаб + отдельные страницы деталей

/ai-cabinet — список кампаний + кнопка "Создать". Клик по кампании → /ai-cabinet/:id — детали с метриками, креативами, управлением.

Как у конкурента: дашборд со списком → детали кампании.

Плюсы

  • Чистый UI, каждая страница фокусирована
  • Детали кампании не перегружают список
  • Масштабируется — легко добавлять секции

Минусы

  • Больше навигации (назад-вперёд)
  • Создание — отдельная модалка или страница
C

Двухколоночный layout

Левая колонка: список кампаний (узкий). Правая: детали выбранной кампании или форма создания.

Master-detail паттерн — как почтовый клиент.

Плюсы

  • Список и детали видны одновременно
  • Быстрое переключение между кампаниями
  • Модный паттерн (Slack, email клиенты)

Минусы

  • Не работает на мобильных
  • Мало места для деталей кампании
  • Сложнее показать полную аналитику

💡 Рекомендация: B — хаб + отдельные страницы. Масштабируется лучше всего, чистый UI, как у конкурента.
